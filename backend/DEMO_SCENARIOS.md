## Data Relationships

### Entity Hierarchy
```
User
  ‚îî‚îÄ DocumentArtifact (baseline document)
       ‚îî‚îÄ Baseline (extracted requirements)
            ‚îî‚îÄ DraftTimeline (structured plan)
                 ‚îî‚îÄ CommittedTimeline (active timeline)
                      ‚îî‚îÄ TimelineStage (4 stages per timeline)
                           ‚îî‚îÄ TimelineMilestone (multiple per stage)
                                ‚îî‚îÄ ProgressEvent (linked to milestones)
  ‚îî‚îÄ JourneyAssessment (periodic self-assessments)
  ‚îî‚îÄ AnalyticsSnapshot (aggregated metrics, linked to timeline version)
```

### Version Tracking
- **Draft Timeline**: Source of truth for timeline structure
- **Committed Timeline**: Immutable snapshot with `version_number` and `draft_timeline_id`
- **Analytics Snapshot**: Linked to timeline via `timeline_version` field

### Temporal Flow
1. User uploads baseline document
2. System extracts requirements ‚Üí Baseline
3. User creates structured timeline ‚Üí DraftTimeline
4. User commits timeline ‚Üí CommittedTimeline (version 1.0)
5. User logs progress ‚Üí ProgressEvent
6. User completes assessments ‚Üí JourneyAssessment
7. System generates analytics ‚Üí AnalyticsSnapshot
8. User revises timeline ‚Üí New CommittedTimeline (version 2.0)
9. Cycle continues...
